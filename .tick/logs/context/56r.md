## Architecture
- KV prompts use a naming convention: `versions.<agent-id>.<key>.<implementation>`
- Each agent needs: `_info`, `_model_name`, `_temperature`, `system`, and `prompt` entries
- FO-04 has two prompt types: `prompt_initial` and `prompt_with_feedback` for the iterative swipe feedback loop
- FO-07 will NOT need `prompt_with_feedback` since there's no iterative feedback loop

## Conventions
- All prompts use Liquid template syntax ({{ variable }}, {% for %}, etc.)
- _info entries have: name, text, author, createdAt fields
- _model_name and _temperature store their value in a `text` field
- system and prompt entries store content in a `text` field
